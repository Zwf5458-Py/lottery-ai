# 台灣威力彩 (Taiwan Weilitsai) 預測模塊設計方案

## 1. 概述
在現有六合彩預測系統中，新增「台灣威力彩」預測板塊。保持原有預測算法和架構，抓取台灣彩券官網最近 5 年歷史數據。
威力彩規則：
- 第一區：01 ~ 38 任選 6 個號碼。
- 第二區：01 ~ 08 任選 1 個號碼。

## 2. 數據庫與數據獲取
### 2.1 數據庫 (`data/lottery.db`)
復用 `lottery_history` 表：
- `lottery_type`: 標識為 `weilitsai`
- `n1` ~ `n6`: 存儲第一區的 6 個開獎號碼 (01-38)
- `special`: 存儲第二區的 1 個開獎號碼 (01-08)

### 2.2 爬蟲腳本 (`data/fetch_weilitsai.py`)
- 目標 URL: `https://www.taiwanlottery.com/lotto/history/history_result/`
- 技術棧: `requests` + `BeautifulSoup` (或 `pandas.read_html`)
- 邏輯: 模擬表單/API請求，分頁或按年月抓取近 5 年數據，清洗並寫入資料庫。

## 3. 核心預測算法與 AI 適配
### 3.1 統計引擎 (`modules/statistics_engine.py`)
移除生肖和波色邏輯，拆分區域進行分析：
- **第一區 (1-38)**: 基於 `n1`~`n6` 分析冷熱、大小 (臨界值 19.5)、單雙、尾數。
- **第二區 (1-8)**: 基於 `special` 分析冷熱、大小 (臨界值 4.5)、單雙。
- **高級算法**: 將馬爾可夫鏈和貝葉斯推斷適配到通用特徵（如號碼區間狀態、大小單雙狀態）上。

### 3.2 模擬器與 AI 引擎 (`modules/simulator.py`, `modules/ai_engine.py`)
- **模擬出號**: 並行計算兩次權重。第一區從 1-38 中按權重選 6 碼（不重複）；第二區從 1-8 中按權重選 1 碼。
- **Prompt 工程**: 為 `weilitsai` 新增專屬 Prompt 模板，強調「38選6 + 8選1」的規則，避免產生生肖/波色相關的幻覺，要求 AI 針對兩區分別進行邏輯推理。

## 4. UI 介面與 API 設計
### 4.1 後端 API (`blueprints/api.py`, `app.py`)
所有數據讀取和模擬 API 均支援 `lottery_type` 參數，動態返回六合彩或威力彩的資料。

### 4.2 前端適配 (`templates/index.html`, `static/js/app.js`)
- **彩種切換**: 增加「六合彩 / 威力彩」的全局切換開關。
- **圖表適配**: 
  - 當選擇威力彩時，隱藏生肖與波色專屬圖表。
  - 通用圖表（冷熱、大小、單雙等）支援展示第一區和第二區的數據（可通過按鈕切換區域，或分區展示）。
- **結果展示**: AI 報告中明確區分「第一區預測（6碼）」與「第二區預測（1碼）」。
