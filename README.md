# 🎰 六合彩智能分析系统

> 基于多维统计引擎 + AI 大模型的全栈轻量级彩票走势分析工具

> [!IMPORTANT]
> **免责声明**：本程序仅供统计学研究与学习交流。彩票为随机事件，AI 模型无法预测未来开奖结果。请理性对待，严禁非法博彩。

## 📖 项目简介

本系统深度聚焦澳门六合彩**特码（Special Number）**的多维度研究，集成了工业级统计引擎与多家 AI 大模型平台，通过加权算法 + AI 推理报告为用户提供可视化的走势分析。

### ✨ 核心亮点

- 🧠 **多平台 AI 引擎**：支持 Google Gemini、Nvidia NIM、OpenAI、本地模型等 10+ 平台一键切换
- 🎰 **双板块独立分析**：隔离支持 **新澳门六合彩**（含生肖、波色特码属性）与 **台湾威力彩**（双区“6+1”模式，38个数字独立分析）两大主流彩种
- 📊 **10+ 个号码与属性统计维度**：大小、单双、冷热、尾数、波色、生肖路单、马尔可夫链、贝叶斯推断、神经网络 (MLP) 拟合
- 🎯 **底层数学引擎出号**：号码由加权模拟器精准计算，AI 负责撰写逻辑自洽的推理报告
- 📸 **一键存图分享**：分析结果支持高清截图下载，自带水印
- ⚙️ **双板块期数独立设置**：支持两彩种独立图表统计期数设置，AI 预测和模拟开奖期数自适应关联
- 👥 **多用户权限体系**：管理员 / VIP / 普通用户 / 试用，积分制图表分析

## 🧠 AI 推算引擎与号码解耦

系统针对不同彩种实现了在**生肖/波色属性级**（六合彩）与**独立球号级**（威力彩）的完全算法解耦：

### 🕸️ 马尔可夫链状态转移 (Markov Chain)
- **新澳门六合彩**：统计特码生肖与波色的状态接替规律，构建 12 生肖和 3 波色的跃迁概率矩阵并转化为模拟权重。
- **台湾威力彩**：脱离生肖，直接构建针对 **1 区 38 个数字的条件转移矩阵**，统计上期 6 正码在历史下期中对 1-38 各号码的条件跃迁频次，计算转移概率。

### ⚖️ 贝叶斯极值反弹 (Bayesian Inference)
- **新澳门六合彩**：结合生肖的历史最大遗漏值，当某生肖遗漏逼近极限时赋予指数级暴增的后验权重。
- **台湾威力彩**：直接为 **1 区 38 个数字独立建立贝叶斯后验概率分布模型**，结合各号码的历史最大遗漏与先验开出率，推算后验反弹概率。

### 🧠 神经网络特征拟合预测 (MLP / LSTM)
- **新澳门六合彩**：将特码生肖走势放入 LSTM 时间序列网络，寻找隐藏的周期波形。
- **台湾威力彩**：提取 38 个号码在不同历史周期中的开出频次及当前遗漏值作为特征，构建 **多层感知机 (MLP) 拟合网络分类器**，输出 38 个号码在下期的概率评分及操作信号。

### 📉 和值动量及 RLE 连涨防跌
- **新澳门六合彩**：探测大小/单双连庄（长龙）是否接近历史极限，在逼近历史天花板（约 13 期）时提升反跳权重。
- **台湾威力彩**：首创针对 **1 区正码和值（Sum Value）进行 RLE 趋势建模**。在和值波动触发“连涨/连跌”反转概率时发出长龙预警，动态调节大小号（1-19 与 20-38）的模拟出球权重。

## 🚀 快速启动

### 方式一：Windows 本地开发（推荐）

```powershell
# 1. 克隆项目
git clone <仓库地址>
cd 六合彩

# 2. 创建虚拟环境
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量
copy .env.example .env
# 编辑 .env 文件，填入你的 API Key

# 5. 启动服务
python app.py
```

访问：`http://127.0.0.1:5000`

### 方式二：Linux 本地开发

```bash
# 环境准备
sudo apt update && sudo apt install -y python3.12-venv

# 启动
cd /path/to/六合彩
bash start.sh dev
```

访问：`http://127.0.0.1:5000`

### 方式三：Docker 生产部署

```bash
sudo docker compose -p macau-mark-six up --build -d
```

### 常用运维命令

```bash
bash logs.sh error    # Gunicorn 错误日志

bash logs.sh ai       # AI 故障日志
bash logs.sh all      # 全部日志

curl http://127.0.0.1:5000/healthz   # 健康检查
```

## ⚙️ AI 平台配置

系统支持多种 AI 平台，在**系统设置 → AI 模型配置**中切换：

| 平台 | 说明 | 模型示例 |
|:-----|:-----|:---------|
| **Google Gemini** | 默认平台 | `gemini-2.5-pro`、`gemini-2.5-flash` |
| **Nvidia NIM** | 国内可达 | `z-ai/glm4.7`、`meta/llama-4-scout-17b-16e-instruct` |
| **OpenAI** | 需科学上网 | `gpt-4o`、`gpt-4o-mini` |
| **本地模型** | 离线可用 | 任意 OpenAI 兼容接口 |
| **自定义平台** | 支持扩展 | 在设置面板中添加任意平台和模型 |

### 环境变量配置

```env
# .env 文件
FLASK_SECRET_KEY=your-secret-key-here
GEMINI_API_KEY=your-gemini-key           # Google Gemini
```

> [!TIP]
> 也可以在登录后的**系统设置面板**中直接填写 API Key 和选择平台，无需编辑文件。Nvidia 平台的 Key 以 `nvapi-` 开头时会自动路由。
