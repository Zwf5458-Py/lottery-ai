## 🧠 3. 威力彩 1区高阶算法号码化重构细节

原系统高阶算法（马尔可夫、贝叶斯、LSTM等）在特码维度上与“生肖/波色”强绑定。已彻底将威力彩的 1区（38个数字）算法解耦并进行号码级重写：

### 3.1 🕸️ 1区号码马尔可夫转移矩阵
* **业务逻辑**：不依靠生肖转移，改为统计相邻两期一区 6个正码号码的跃迁转移。
* **数学模型**：构建 $38 \times 38$ 的跃迁状态转移频次矩阵 $T$，其中 $T_{i, j}$ 表示历史上号码 $i$ 在当前期开出时，下期开出号码 $j$ 的频次。
* **平滑与预测**：当预测下期时，提取最新一期开出的 6个号码 $\{n_1, n_2, \dots, n_6\}$ 作为状态输入，累加跃迁概率并进行拉普拉斯平滑（$\alpha=1$），最终转化为号码级的转移权重分配。

### 3.2 ⚖️ 1区号码贝叶斯后验反弹模型
* **业务逻辑**：直接为一区 38个号码分别建立贝叶斯推断模型。
* **数学模型**：以号码的当前连续遗漏期数 $x$ 作为新证据，结合历史最大遗漏值 $x_{max}$ 计算似然度，修正其下期开出的后验概率 $P(A|B)$。当某个球在极限遗漏 80% 以上时，自动触发极限警报，权重呈指数级调升（高达 $3.0\times$ 乘数）。

### 3.3 🧠 1区号码 MLP 神经网络拟合网络
* **业务逻辑**：放弃原来的 LSTM 时序生肖拟合，为 38个数字独立特征工程，使用 `MLPClassifier` 多层感知机分类器拟合。
* **特征矩阵**：提取最近 50 期、10 期内各号码出现的频次，当前连续遗漏值等 5 维数据，输入三层感知机中进行权重拟合，输出 1-38 对应号码的下期概率预测评分（0-100）。

### 3.4 📉 1区正码和值动量反演（和值路单 RLE）
* **业务逻辑**：对最近 150 期的 1区正码累加和值（和值最活跃区间为 $[85, 145]$）进行 RLE 行程编码，统计和值波动的连涨或连跌长龙。
* **出号约束**：当和值长龙触发拐点反转概率（防跌/防涨警告）时，后端自动降低大号码（20-38）或小号码（1-19）的出球加权概率，确保模拟器在宏观上完美遵从和值动量反演。

### 3.5 📊 高阶衍生统计算法（极距、三分比、泊松同尾、AC复杂度）
为了向用户和 AI 提供更高维度的分析视角，系统新增了 4 种独立的高维特征刻画模型：
* **极距跨度与均值回归**：计算历史每期最大开出号与最小号的差值（Span），基于中心极限定理，跨度呈现类高斯分布。当极距长时间偏离期望均值时，预测其产生均值回归现象。
* **三分比与质合分布**：将号码均等分为3个区间，并统计区间号码比例以及质数与合数比值。用于预警极端偏态分布。
* **泊松同尾分布图**：统计每期出球中“相同尾数”（如同尾4：14、24、34）的出现次数，使用泊松分布模型推演下期同尾极值重叠的概率。
* **AC值（数字复杂度）模型**：统计号码组中所有可能差值的非重复数量。AC值越高，表示该组号码之间的等差结构越低（越随机）。系统通过散点图与历史趋势推算“安全的高频AC区间”。

### 3.6 叠加出号整合机制
在模拟开奖出号（`modules/simulator.py` 中的 `simulate_draws`）时，系统将上述 4 种独立权重相乘，并应用**动态冷温热三态池分配**（热 30%、温 40%、冷 30% 限制出号）和 **AC值过滤（AC $\ge 4$）**。

## ⚙️ 4. 统计期数多板块物理隔离设置设计

为了实现两个彩种在“图表统计期数”和“AI模型分析期数”上的完全隔离，我们对配置层进行了接口映射化重构：

### 4.1 数据库嵌套结构设计
在 SQLite 用户独立库的 `user_settings` 表中，键 `chart_periods` 对应的 value 被规范化升级为嵌套的分支字典：
```json

  "weilitsai": {
    "zodiac_trend": 100,

    "big_small": 150,
    "markov": 150,
    "hot_cold": 60,
    "tail": 80,
    "bayesian": 90,
    "lstm": 90,
    "ai_raw_data": 150
  }
}
```

### 4.2 配置管理器兼容提取与 AI 关联
* 后端配置管理模块 [config_manager.py](file:///Users/zhuangwanfu/code%20/%E5%85%AD%E5%90%88%E5%BD%A9/modules/config_manager.py) 中，`get_chart_periods(user_id=None, lottery_type: str = 'macaujc')` 根据当前的彩种获取对应分支的数据。若旧配置为平铺格式，则自动升级为双分支结构，实现 100% 向后兼容。
* `modules/ai_engine.py` 及模拟器在读取 `ai_raw_data` 期数进行 Prompt 拼接时，全部传入 `lottery_type`，实现了 **“AI大模型分析时数据期数自动关联至当前所在板块”**。

### 4.3 接口动态平铺转换
* **GET /api/settings**：读取 `?type` 参数，提取相应彩种的期数分支，并以平铺格式（平坦的 periods 字典）返回给前端。
* **POST /api/settings**：接收平铺的数据，读取 `?type` 参数定位当前彩种，在执行扣积分校验后，将其更新到总的隔离字典对应的子项下，最后保存回数据库。
* 这使得前端 JS / HTML 业务无需对大段表单进行重构，依然以最直接的平铺数据绑定交互。

## 7. 最新 UI 优化与 Bug 修复记录 (2026-06-15)

在最近一次迭代中，针对用户在使用过程中反馈的 UI 交互与图表视觉问题，进行了全方位的深度优化及 Bug 修复：

### 7.1 📊 图表与卡片布局优化
* **三分比与质合形态分析卡片**：将“三分比”与“质合形态”调整为左右对齐的并排布局，修复了“形态不见了”的渲染 Bug，提升视觉美感。
* **特码生肖路单 UI**：增大了 12 生肖路单的高度间距（垂直行高由默认调整至 `ROW_H = 34`），去除了底部无用的年份及坐标轴空间，使卡片整体上下分布更舒展。
* **路单开奖日期联动悬停**：移除了冗余的年份显示，在路单的每个开奖球上增加了透明鼠标触碰感应区。当鼠标触碰到或点击开奖球时，Tooltip 会实时显示该期开奖的完整日期。

### 7.2 🕸️ 马尔可夫链文本重叠修复
* **文字遮挡与重叠问题**：优化了极坐标图 (PolarArea) 与雷达图的缩放比例及 padding。通过动态调大上限值（`maxVal + 0.6`）以及调整标签间距，彻底解决了高频出号色块遮挡生肖/号码文字的问题。

### 7.3 ✨ 号码频率分布图“闪烁星星”精准定位与渲染
* **星星位置与缺失修复**：修复了原本只有前几个号码有星星、后面没有星星的 Bug，确保高频号码上方正确标注 ✨ 符号。
* **渲染与动画机制重构**：为了解决 Chart.js 内部使用 `scatter` (数值轴) 与 `bar` (分类轴) 混用时散点坐标错位、Canvas 内部无法制作复杂 CSS 动画的痛点，移除了旧有散点数据集，改为通过 Chart.js 插件机制（`afterRender` 钩子）动态计算 Canvas 像素坐标并向页面注入真实 HTML 元素。
* **对比度与闪烁特效**：结合 `@keyframes star-blink` CSS 动画，为星星添加了高对比度呼吸闪烁效果，显著提升了视觉清晰度。

### 7.4 🐛 后端 500 报错与缓存清理
* **Numpy 序列化与变量缺失修复**：修复了 `/api/statistics?type=macau` 接口由于卡方检验中 `_chi_square_p_value` 变量缺失以及 `numpy.bool_` 无法直接进行 JSON 序列化导致接口报错 500 的 Bug。
* **FileSystemCache 死锁解决**：由于 Flask-Caching 会将 500 报错的 JSON 也进行持久化缓存，导致修复后页面依然报错。清空了 `data/cache/` 目录并明确了清除缓存的操作指南（参见 6.4 节）。
